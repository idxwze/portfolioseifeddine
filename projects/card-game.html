C++ Card Game

An object-oriented card game in C++ with clean architecture, game rules engine, and structured design.

C++ OOP CLI Design

Overview

This project demonstrates strong object-oriented design using C++: classes, encapsulation, inheritance, and polymorphism, with a clear separation between game logic, player actions, and I/O.

The goal was to build a maintainable game core that is easy to extend (new card types, rules, or game modes).

Problem

  • Design a card game with clear rules and predictable game flow.
  • Keep the codebase extensible (new cards/rules) without rewriting everything.
  • Avoid spaghetti logic by structuring responsibilities properly.

Solution

I designed the game using an OOP architecture: a base Card type, specialized derived cards, and a Game engine that controls turns, state, and rule validation.

  • Core entities: Game, Player, Deck, Hand, Card (base) + derived cards.
  • A rules/validation layer to ensure legal moves and consistent state updates.
  • Separated I/O from logic (so you can swap CLI later if needed).

Key features

Clean game loop

Turns, phases, and end conditions handled by a single game engine.

OOP extensibility

Add new card types without changing existing logic (polymorphism).

Validation & state safety

Moves are validated to avoid invalid states and edge case bugs.

Readable structure

Clear folder/class responsibilities and documentation-friendly flow.

Architecture

High-level structure: 

/src
  Game.cpp / Game.h
  Player.cpp / Player.h
  Deck.cpp / Deck.h
  Hand.cpp / Hand.h
  Card.h
  cards/
    AttackCard.h
    DefenseCard.h

The game engine controls the flow; cards encapsulate behavior; players interact through validated actions.
