e-Hotels Reservation Platform

Database-driven reservation backend with schemas, constraints, triggers, indexing, and a dynamic web UI.

SQL ERD Constraints Triggers Indexes Web UI

Overview

e-Hotels is a database-first project where customers can search and reserve hotel rooms with real-time availability, and employees can convert reservations into rentals at check-in (or create rentals directly).

The project required: an ER diagram, relational schema, integrity constraints, SQL queries + triggers, indexes, and a web application with filters + admin CRUD screens.

Problem

  • Model a realistic multi-chain hotel system (chains ‚Üí hotels ‚Üí rooms) with strict dependencies.
  • Support both reservation and rental flows, including walk-in rentals without prior reservation.
  • Maintain archives of reservations/rentals even if rooms/hotels are deleted later.

Key requirements

  • Store chains (HQ address, contacts, number of hotels), hotels (category, contacts), rooms (price, amenities, capacity, view, extendable, damages).
  • Support customers + employees (roles). Each hotel must have a manager.
  • Search/filter rooms by date range, capacity, area, chain, hotel category, hotel size, and price. Results update when filters change.
  • Provide two SQL Views: (1) available rooms by area; (2) capacity of rooms for a specific hotel.

(Summarized from the project brief; full specification in the original PDF.)

Solution

I designed the system with a normalized relational model and an integrity layer (PK/FK/domain/user constraints), then added triggers + queries to enforce business rules and ensure consistent reservation/rental transitions.

  • Normalized schema for Chains ‚Üí Hotels ‚Üí Rooms + link tables for amenities and roles.
  • Triggers to validate booking/rental rules and keep state consistent.
  • Indexes chosen to accelerate availability searches and common filter queries.
